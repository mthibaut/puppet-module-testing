Gemfile

Describes ruby dependencies (not puppet modules!)

source :rubygems

group :development, :test do
  gem 'puppetlabs_spec_helper', :require => false
  gem 'rspec'
  gem 'rspec-puppet'
end

if puppetversion = ENV['PUPPET_GEM_VERSION']
  gem 'puppet', puppetversion, :require => false
else
  gem 'puppet', :require => false
end

Write some tests

  • rspec-puppet does parsing only
  • Provide the facts you need (e.g. $::osfamily)
  • Filename must match spec/*/*_spec.rb
  • Follow naming conventions
    • class autofs => spec/classes/autofs_spec.rb
    • define ntp => spec/defines/ntp_spec.rb
    • function hash_to_list() => spec/functions/hash_to_list_spec.rb
  • Create a new file or add to existing one for each new bug

Look at the manifests/init.pp file

class autofs (
  $owner   = $::autofs::params::owner,
  $group   = $::autofs::params::group,
  $mode    = $::autofs::params::mode,
  $master  = $::autofs::params::master,
  $package = $::autofs::params::package,
  $service = $::autofs::params::service
  #$mounts is NOT used here! They are used in autofs::loader with hiera_hash
) inherits autofs::params {

  include autofs::package
  include autofs::config
  include autofs::service
